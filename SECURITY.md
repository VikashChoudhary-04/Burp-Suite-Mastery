# Security Policy

- Thank you for helping improve the security and quality of **Burp Suite Mastery**.

- This repository is an educational resource focused on authorized web application security testing using Burp Suite.

---

## Reporting Security Issues

- If you discover a security issue related to this repository (such as unsafe example code, exposed sensitive information, or incorrect security guidance), please report it responsibly.

- You can:

  * Open a GitHub Issue (for non-sensitive problems).
  * Contact the repository maintainer directly if the issue should not be disclosed publicly.

- Please include:

  * A clear description of the issue.
  * Steps to reproduce (if applicable).
  * The affected file(s).
  * Any suggested improvements.

---

## Scope

- This security policy applies only to:

  * Repository content
  * Documentation
  * Example code
  * Diagrams
  * Educational resources

- It does **not** apply to:

  * Third-party software
  * Burp Suite itself
  * External websites
  * Practice labs
  * Vulnerability disclosure programs

- Please report issues in those projects through their official security channels.

---

## Responsible Disclosure

- If an example, exercise, or workflow could unintentionally encourage unsafe or unauthorized testing, please report it.

- The goal of this repository is to promote:

  * Ethical security research
  * Authorized penetration testing
  * Responsible vulnerability disclosure
  * Professional investigation methodology

---

## Educational Purpose

- All examples in this repository are intended for learning.

- Only perform security testing against systems that:

  * You own, or
  * You have explicit written permission to assess.

- Unauthorized testing may be illegal and unethical.

---

## Supported Versions

- The project is under active development.

- The latest version on the default branch is the supported version.

| Version        | Supported |
| -------------- | :-------: |
| Latest         |     ✅     |
| Older releases |     ❌     |

---

## Security Best Practices

- When using Burp Suite or following examples from this repository:

  * Test only authorized targets.
  * Keep Burp Suite updated.
  * Verify findings before reporting them.
  * Collect evidence to support conclusions.
  * Avoid assumptions based on a single observation.
  * Protect any sensitive data captured during testing.
