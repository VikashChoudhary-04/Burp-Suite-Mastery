- Sequencer is one of the most misunderstood tools in Burp Suite.

- Many people describe it as a "randomness tester."

- While that description is partially true, it misses the real purpose of the tool.

- Sequencer exists to help security testers evaluate whether security-critical values—such as session identifiers, authentication tokens, password reset tokens, and CSRF tokens—appear sufficiently unpredictable for their intended use.

- Its purpose is not to prove that values are random.

- Its purpose is to help answer a much more important security question:

  > **Could an attacker reasonably predict future values?**

---

## Why Predictability Matters

- Modern web applications rely on unpredictable values to maintain security.

- Examples include:

  * Session identifiers
  * Authentication tokens
  * Password reset links
  * CSRF tokens
  * API keys
  * One-time verification tokens

- If these values can be predicted, attackers may be able to:

  * Hijack active sessions
  * Access another user's account
  * Bypass authentication workflows
  * Reuse or guess reset links
  * Circumvent security controls

- Predictability—not simply randomness—is the security concern.

---

## Random Does Not Mean Secure

- It is important to understand that randomness alone is not enough.

- A value may appear random while still being generated by a weak or predictable process.

- Likewise, a value that looks structured may still be secure if it is produced using a cryptographically strong random source.

- Sequencer helps you gather evidence.

- It does not guarantee security or insecurity.

---

## Where You'll Use Sequencer

- During an assessment, Sequencer is commonly used to evaluate:

  * Session cookies
  * Login tokens
  * Password reset tokens
  * Anti-CSRF tokens
  * API authentication tokens
  * Custom application identifiers

- These values often represent the foundation of an application's authentication and session management.

---

## Investigation Mindset

- Before collecting tokens, ask yourself:

  1. Which value am I evaluating?
  2. Why is this value security-critical?
  3. What would happen if it were predictable?
  4. What evidence would indicate a weakness?

- Sequencer is most useful when you begin with clear investigative questions.

---

## Prediction Challenge

- Suppose an application generates these session identifiers:

  ```text
  A81F02
  A81F03
  A81F04
  A81F05
  ```

- Ask yourself:

  * Do these values suggest a pattern?
  * Would you trust them for session management?
  * What additional evidence would you collect before concluding they are predictable?

- Avoid drawing conclusions from a few examples alone.

---

## Key Takeaways

* Sequencer evaluates the unpredictability of security-critical values.
* Predictability—not just randomness—is the real security concern.
* Statistical analysis supports investigation but does not replace judgment.
* Begin every assessment with a clear hypothesis.
* Evidence should guide conclusions.
