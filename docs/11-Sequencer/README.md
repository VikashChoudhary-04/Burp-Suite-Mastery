# Sequencer

> *"The Burp Suite tool for evaluating the quality and unpredictability of session tokens."*

---

## What is Sequencer?

- Burp Sequencer analyzes the randomness and quality of values generated by an application.

- It is commonly used to assess:

  * Session identifiers
  * Authentication tokens
  * Password reset tokens
  * CSRF tokens
  * API keys
  * One-time tokens
  * Custom identifiers

- Rather than checking whether a value is "random enough" by eye, Sequencer performs statistical analysis to help identify predictable patterns.

---

## Why Learn Sequencer?

- Modern applications rely on unpredictable values to protect users.

- If attackers can predict future tokens, they may be able to:

  * Hijack sessions
  * Bypass authentication
  * Guess password reset links
  * Impersonate users
  * Access protected resources

- Sequencer helps evaluate whether generated values appear sufficiently unpredictable.

## Professional Mindset

- Sequencer is not simply a statistics tool.

- It is an investigation tool.

- Its purpose is to answer one question:

  > **Can an attacker reasonably predict future values?**

- That question is central to secure session management.

## Recommended Learning Strategy

- For every lesson:

  1. Learn the concept.
  2. Observe generated tokens.
  3. Form a hypothesis.
  4. Analyze with Sequencer.
  5. Interpret the results.
  6. Validate your conclusions.

- The goal is not simply to generate statistics.

- The goal is to determine whether security-critical values are sufficiently unpredictable.

---

## What's Different About This Module?

- This module introduces **Prediction Challenges**.

- Instead of asking:

  > "Is this random?"

- You'll practice asking:

  * Could an attacker predict this?
  * What evidence supports that conclusion?
  * Which patterns deserve investigation?
  * What additional testing is required?

- These exercises develop the reasoning needed for evaluating session management and token security.
