# Comparer

> *"The Burp Suite tool for identifying meaningful differences between data."*

---

## What is Comparer?

- Burp Comparer helps security testers identify differences between two pieces of data.

- These may include:

  * HTTP requests
  * HTTP responses
  * Headers
  * Cookies
  * JSON
  * JWTs
  * API responses
  * Plain text
  * Encoded values

- Instead of manually searching for changes, Comparer highlights differences, making investigations faster and more reliable.

---

## Why Learn Comparer?

- Security investigations often involve comparing:

  * Admin vs User responses
  * Successful vs Failed requests
  * Authenticated vs Unauthenticated requests
  * Two JWTs
  * Two API responses
  * Different application states

- Missing a single character can completely change the meaning of a request or response.

- Comparer helps you notice those changes.

## Professional Mindset

- Comparer is not simply a "diff tool."

- It is an investigation tool.

- Its purpose is to answer one question:

  > **What changed, and why does it matter?**

- Professional testers compare evidence before drawing conclusions.

## Recommended Learning Strategy

- For every lesson:

  1. Understand the concept.

  5. Review the Cheat Sheet.
  6. Finish with the Quick Revision.

---

## What's Different About This Module?

- Throughout this module, you'll complete **Difference Challenges**.

- Instead of asking:

  > "Which request is correct?"

- We'll ask:

  * What changed?
  * Which differences matter?
  * Which differences can be ignored?
  * What does the change suggest about the application's behavior?

- These exercises develop the analytical thinking used in professional web application security assessments.
