# Repeater

> *"The most powerful Burp Suite tool for manual web application security testing."*

---

## What is Repeater?

- Burp Repeater allows you to manually send HTTP requests to a server, modify them, resend them, and analyze the responses.

- Unlike your browser, which automatically generates requests, Repeater gives you complete control over every part of an HTTP request.

- For many penetration testers, Repeater is the most frequently used tool in Burp Suite because it enables precise testing and experimentation.

---

## Why Learn Repeater?

- Most web application vulnerabilities cannot be discovered by sending a request only once.

- Security testing requires you to:

  * Modify one parameter
  * Send the request
  * Observe the response
  * Modify another parameter
  * Compare responses
  * Repeat the process

- Repeater was designed specifically for this workflow.

## Why Professionals Love Repeater

- Experienced penetration testers spend a significant portion of their assessment time inside Repeater.

- Repeater is commonly used to investigate:

  * IDOR
  * SQL Injection
  * Cross-Site Scripting (XSS)
  * Business Logic flaws
  * Access Control issues
  * JWT manipulation
  * API security
  * Server-Side Request Forgery (SSRF)
  * XML External Entity (XXE)
  * File upload issues
  * Request smuggling investigations
  * Race condition validation

- Repeater provides the control needed to understand how an application behaves when individual parts of a request change.
