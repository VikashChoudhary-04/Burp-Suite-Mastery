# Burp Scanner

- Burp Scanner is Burp Suite's automated vulnerability assessment engine.

- It combines passive and active security testing to identify potential weaknesses in web applications. Used correctly, it can significantly improve efficiency by discovering issues that would be time-consuming to identify manually.

- However, Scanner is **not a replacement for a penetration tester**.

- Professional testers use Scanner to assist investigations, validate observations, and prioritize manual testing—not to generate reports without verification.

---

## Learning Objectives

- By the end of this module, you will be able to:

  * Understand how Burp Scanner works.
  * Differentiate between passive and active scanning.
  * Decide when automated scanning is appropriate.
  * Interpret Scanner findings critically.
  * Validate reported vulnerabilities manually.
  * Integrate Scanner into a professional testing workflow.

---

## Professional Mindset

- Do not ask:

  > **"What vulnerabilities did Scanner find?"**

- Instead ask:

  > **"What evidence supports Scanner's findings, and how should I validate them?"**

- Automation accelerates discovery.

- Human reasoning confirms reality.
