## What is Burp Scanner?

- Burp Scanner is an automated security testing tool within Burp Suite that helps identify potential vulnerabilities in web applications.

- Its purpose is to:

  * Improve testing coverage.
  * Detect common security issues.
  * Assist penetration testers during assessments.
  * Support—not replace—manual testing.

- Remember:

  - **Scanner identifies possibilities.
Penetration testers establish facts.**

---

## Passive vs Active Scanning

### Passive Scanning

* Observes existing traffic.
* Does not send additional test requests.
* Lower operational impact.
* Suitable during application exploration.

### Active Scanning

* Sends additional requests.
* Tests application behavior.
* Generates more traffic.
* Requires authorization.
* May affect application performance.

## Core Principles

- Always remember:

  * Understand the application before scanning.
  * Define scope carefully.
  * Configure Scanner intentionally.
  * Review evidence—not just severity.
  * Validate findings manually.
  * Report only confirmed vulnerabilities.

## Common Mistakes

- Avoid:

  * Trusting every Scanner finding.
  * Assuming a clean scan means a secure application.
  * Ignoring business context.
  * Scanning outside authorized scope.
  * Leaving scans unattended.
  * Reporting findings without validation.
