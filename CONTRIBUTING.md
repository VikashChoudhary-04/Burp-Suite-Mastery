# Contributing to Burp Suite Mastery

- Thank you for your interest in contributing to **Burp Suite Mastery**.

- The goal of this repository is to provide a structured, practical, and professional learning resource for Burp Suite and web application security.

- Whether you're fixing a typo, improving an explanation, or contributing a new lesson, your help is appreciated.

---

## Repository Goals

- Every contribution should help readers:

  * Understand concepts before memorizing steps.
  * Develop practical Burp Suite skills.
  * Think like professional penetration testers.
  * Learn ethical and authorized testing practices.

---

## Before You Contribute

- Please ensure that your contribution:

  * Is technically accurate.
  * Uses clear and concise language.
  * Follows the existing lesson structure.
  * Supports learning through practical examples.
  * Avoids unnecessary complexity.

---

## Writing Standards

- Every lesson should:

  * Explain **why** before **how**.
  * Introduce one concept at a time.
  * Use professional terminology consistently.
  * Include practical examples where appropriate.
  * Encourage evidence-based investigation.
  * Focus on authorized security testing.

---

## Lesson Structure

- When creating a new lesson, follow the repository's standard structure:

  1. Overview
  2. Learning Objectives
  3. Prerequisites
  4. Difficulty
  5. Estimated Time
  6. Introduction
  7. Why This Exists
  8. Mental Model
  9. Behind the Scenes
  10. Interface Walkthrough
  11. Practical Demonstration
  12. Professional Workflow
  13. Attack Scenario
  14. Field Notes
  15. Common Mistakes
  16. Practice Exercises
  17. Capstone Challenge
  18. Investigation Journal
  19. Interview Questions
  20. Quick Revision
  21. Mastery Checklist
      
---

## Markdown Guidelines

* Use descriptive headings.
* Keep paragraphs short.
* Prefer bullet lists where appropriate.
* Use fenced code blocks for examples.
* Use tables only when they improve clarity.
* Use GitHub callouts for important notes.

Example:

```md
> [!TIP]
> Change one variable at a time.
```

---

## Images

- When adding images:

  * Use high-resolution screenshots.
  * Remove sensitive information.
  * Annotate screenshots when helpful.
  * Store shared images in the `assets/` directory.
  * Give files descriptive names.

Example:

```text
repeater-interface.png
request-lifecycle.png
response-analysis.png
```

---

## Code Examples

- Code examples should:

  * Be easy to understand.
  * Use safe demonstration values.
  * Avoid unnecessary complexity.
  * Support the lesson objective.

---

## Exercises

- Exercises should:

  * Reinforce concepts.
  * Encourage observation.
  * Promote structured investigation.
  * Never encourage unauthorized testing.

---

## Commit Messages

- Use clear, descriptive commit messages.

- Examples:

  ```text
  Add Repeater interface lesson

  Improve response analysis examples

  Fix typo in Proxy module

  Add screenshots for Target module
  ```

---

## Pull Requests

- Before submitting a pull request, confirm that:

  * [ ] The content has been proofread.
  * [ ] Markdown formatting is correct.
  * [ ] Technical information has been verified.
  * [ ] Examples are clear and accurate.
  * [ ] Images (if any) are properly referenced.
  * [ ] The lesson follows the repository structure.

---

## Code of Conduct

- Please:

  * Be respectful.
  * Be constructive.
  * Welcome feedback.
  * Focus on improving the educational quality of the repository.

---

## Reporting Issues

- If you find:

  * Technical inaccuracies
  * Broken links
  * Typographical errors
  * Outdated screenshots
  * Unclear explanations

please open an issue with as much detail as possible.

---

## Educational Scope

- This repository is intended for **authorized security testing and education**.

- Contributions should not encourage unauthorized access, illegal activity, or unsafe practices.

---

## Thank You

- Thank you for helping improve **Burp Suite Mastery**.

- Every improvement makes this resource more valuable for learners around the world.
